# Building Reusable React Components

Reusable components improve maintainability and scalability in React apps.

## Best Practices

- **Single Responsibility**: Each component should have one job.
- **Props for Flexibility**: Use props to customize behavior.
- **TypeScript**: Add types for better developer experience.

## Example: Button Component

```tsx
interface ButtonProps {
  label: string;
  onClick: () => void;
}

export function Button({ label, onClick }: ButtonProps) {
  return <button onClick={onClick}>{label}</button>;
}
```

## Usage

```tsx
<Button label="Click Me" onClick={() => alert('Clicked!')} />
```
