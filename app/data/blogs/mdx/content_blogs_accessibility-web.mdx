# Building Accessible Web Apps

Accessibility (a11y) makes your site usable for everyone, including those with disabilities.

## Best Practices

- **Semantic HTML**: Use proper tags like `<nav>`, `<main>`.
- **Keyboard Navigation**: Ensure all interactive elements are keyboard-accessible.
- **ARIA**: Add ARIA attributes where needed.

## Example: Accessible Button

```tsx
<button aria-label="Submit form" onClick={handleSubmit}>
  Submit
</button>
```

## Testing

Use [WAVE](https://wave.webaim.org/) to check for accessibility issues.
